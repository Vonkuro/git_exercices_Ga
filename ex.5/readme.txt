Vous venez de faire un 'git fetch' et vous vous appercevez qu'un dev a 
modifié exactement le même fichier à la même ligne sur laquelle vous venez 
de travailler.

Vous voulez sauvegarder l'état de votre espace de travail.

Le confilt semble inévitable.

- Commiter avec la nouvelle version de first.txt
- Utiliser 'git pull' (ou git merge origin/master) pour merger origin/master
dans votre branche master
- Résoudre le confilt dans first.txt
- Commiter.
